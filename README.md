# 基于多服务集成的旅游模糊需求精准转化及行程优化系统

基于 AI 的长沙旅游智能规划系统，支持自然语言输入旅行需求，自动生成个性化行程方案。

## 技术栈

| 层级 | 技术 |
|---|---|
| 前端 | Vue 3 + Vite + Element Plus + Pinia + Vue Router |
| 后端 | Node.js + Express |
| 数据库 | MySQL 8.0 |
| AI 引擎 | 通义千问 (Qwen) API - 阿里云百炼平台 |
| 地图服务 | 高德地图 POI 搜索 / 天气 / 路线规划 |
| 天气服务 | 和风天气 API |

## 功能特性

### 核心功能
- **AI 自然语言规划**：支持中文模糊需求描述（如"情侣两天游，预算1000，想看博物馆和吃小吃"），自动解析并结构化
- **智能行程生成**：AI 生成完整行程（景点/餐厅/酒店/交通），含坐标验证、费用计算、路线优化
- **多轮对话调整**：支持"换一批""加个博物馆""便宜点酒店"等自然语言修改
- **TSP 路线优化**：最近邻贪心算法，每日景点最优游览顺序
- **交通方式推荐**：综合距离/天气/预算的智能推荐（地铁/公交/打车/步行）
- **热度预测与避峰**：多因子加权预测各时段拥挤度，推荐最佳游览时间

### 用户系统
- 邮箱注册 / 登录（含邮箱验证码）
- 游客免登录体验模式
- 用户偏好画像学习（行为采集 → 加权画像 → 个性化推荐）
- 行程收藏与反馈

### 数据服务集成
- **高德地图**：POI 搜索、坐标验证、周边设施查询
- **和风天气**：实时天气 + 未来5天预报，雨天自动推荐室内景点
- **通义千问**：行程生成、对话交互、换一批推荐

## 快速开始

### 环境要求
- Node.js >= 18
- MySQL >= 8.0
- npm >= 9

### 安装依赖
```bash
npm run install:all
```

### 配置环境变量
```bash
cp .env.example .env   # 复制模板
# 编辑 .env 填入真实 API 密钥
```

需配置以下密钥：
| 变量 | 说明 | 获取地址 |
|---|---|---|
| `QWEN_API_KEY` | 通义千问 API Key | [阿里云百炼](https://bailian.console.aliyun.com) |
| `AMAP_KEY` / `AMAP_WEB_KEY` / `AMAP_JS_KEY` | 高德地图密钥 | [高德开放平台](https://lbs.amap.com) |
| `QWEATHER_KEY` | 和风天气 Key | [和风天气](https://dev.qweather.com) |
| `DB_PASSWORD` | 数据库密码 | 本地 MySQL |

### 启动开发环境
```bash
npm run dev          # 同时启动前后端 (server:3002, client:5173)
```

单独启动：
```bash
npm run dev:server   # 仅后端 :3002
npm run dev:client   # 仅前端 :5173
```

### 生产部署
```bash
npm run build:client    # 构建前端到 client/dist
npm start:server        # 启动后端（Express 自动托管静态文件）
```

## 已知 Bug 与限制

> ⚠️ **本项目为毕业设计作品，部分功能存在已知问题，仅供演示参考。**

### 1. 坐标验证偏差
**现象**：高德 POI 搜索可能返回跨省错误结果

**原因**：高德 API 的 `citylimit` 参数非硬性约束，当关键词在指定城市无精确匹配时会扩展到其他省市。

**当前处理**：已添加 Haversine 距离校验（30km阈值），偏差过大时回退使用 AI 生成的原始坐标。

### 2. 高德 API 限流 (CUQPS)
**现象**：高频调用时出现 `API限流 CUQPS` 警告，导致部分搜索结果缺失或延迟。

**原因**：高德免费版 QPS 限制较低（Web服务约50次/秒），批量搜索时容易触发限流。

**当前处理**：已实现并发限速器 (`ConcurrencyLimiter`) + 自动重试机制（最多3次），但极端情况下仍可能丢失数据。

### 3. 用户行为记录偶发失败
**现象**：日志中出现 `记录行为失败: userId is not defined` 错误。

**原因**：函数签名参数不一致（历史代码遗留），已在本次修复中解决。

### 4. AI 生成结果的稳定性
**现象**：
- AI 返回的 JSON 格式偶尔不规范（中文标点/缺少逗号等），依赖后端的 `fixJSON()` 容错修复
- 景点价格/评分等信息由 AI 凭知识库生成，可能与实际情况有出入
- 同一需求多次请求，生成的行程差异较大（无确定性保证）

### 5. 数据库未做连接池健康检查
**现象**：长时间运行后可能出现 MySQL 连接断开导致的查询失败。

**建议**：生产环境应增加连接池心跳检测与自动重连机制。

### 6. 会话管理仅内存存储
**现象**：服务重启后所有会话状态丢失，多实例部署时无法共享会话。

**建议**：生产环境应改用 Redis 存储会话状态。
